# Shelly i3 (Gen1) — Tam API Komut Listesi

**Model:** SHIX3-1
**Nesil:** Gen1
**Özellikler:** **3 kanallı kuru kontak girişi** — röle YOK, sadece input. Klasik anahtar/butonları akıllı butona çevirmek için.
**Voltaj:** 12V veya 24-60V AC/DC
**Kullanım:** Mevcut ev anahtarlarını "akıllı" yapmak — anahtar fiziksel olarak çalışır ama event de tetikler

> i3 **çıkış vermez**. Sadece input event'leri üretir ve webhook/MQTT olarak başka cihazlara gönderir.

---

## 1. Cihaz Bilgileri

```
GET http://192.168.1.42/shelly
```

**Cevap:**
```json
{
  "type": "SHIX3-1",
  "mac": "...",
  "auth": false,
  "num_outputs": 0,
  "num_meters": 0,
  "num_inputs": 3
}
```

## 2. Input Durumu Sorgulama

### Input 0 durumu
```
GET http://192.168.1.42/input/0
```

**Cevap:**
```json
{
  "input": 0,
  "event": "S",        // S=short, L=long, SS=double, SL=short-long, ...
  "event_cnt": 12
}
```

### Input 1 ve 2 durumu
```
GET http://192.168.1.42/input/1
GET http://192.168.1.42/input/2
```

### Tüm durum
```
GET http://192.168.1.42/status
```

---

## 3. Event Tipleri

i3 farklı buton hareketlerini tanır:

| Event | Açıklama |
|---|---|
| `S` | Short push (kısa bas) |
| `SS` | Double short push (çift kısa) |
| `SSS` | Triple short push |
| `L` | Long push (uzun bas) |
| `SL` | Short → Long (kısa sonra uzun) |
| `LS` | Long → Short |

---

## 4. Buton Tipi Konfigürasyonu

### Input modu
```
GET http://192.168.1.42/settings/input/0?btn_type=momentary
GET http://192.168.1.42/settings/input/0?btn_type=toggle
GET http://192.168.1.42/settings/input/0?btn_type=edge
GET http://192.168.1.42/settings/input/0?btn_type=detached
```

| Tip | Davranış |
|---|---|
| `momentary` | Bas-bırak (event üretir, S/SS/L/SL desteği) |
| `toggle` | Klasik anahtar (her kenar event) |
| `edge` | Kenar event'i |
| `detached` | Sadece event üretir, başka şey yok |

### Buton ters çevir
```
GET http://192.168.1.42/settings/input/0?btn_reverse=1
```

### Long-push tetik süresi (ms)
```
GET http://192.168.1.42/settings?longpush_duration_ms_max=800
GET http://192.168.1.42/settings?longpush_duration_ms_min=300
```

## 5. Webhook / Actions (Asıl Özellik)

i3'ün gücü buradadır — event'lere göre HTTP/MQTT tetikle.

### Action isimleri (her input için)
- `btn_on_url` — input HIGH oldu (örn. anahtar açıldı)
- `btn_off_url` — input LOW oldu
- `shortpush_url` — kısa bas event
- `longpush_url` — uzun bas event
- `double_shortpush_url` — çift kısa
- `triple_shortpush_url` — üçlü kısa
- `shortpush_longpush_url` — short → long
- `longpush_shortpush_url` — long → short

### Örnek: input 0 kısa basıldığında URL çağır
```
GET http://192.168.1.42/settings/actions?index=0&name=shortpush_url&enabled=true&urls[]=http://YOUR-SERVER/light-toggle
```

### Birden fazla URL aynı event için
```
GET http://192.168.1.42/settings/actions?index=0&name=shortpush_url&enabled=true&urls[]=http://server1/a&urls[]=http://server2/b
```

## 6. MQTT Topic'leri

```
shellies/shellyi3-XXX/input/0          — durum (0/1)
shellies/shellyi3-XXX/input/0/event    — son event (S, L, SS, ...)
shellies/shellyi3-XXX/input/0/event_cnt — event sayacı
shellies/shellyi3-XXX/input_event/0    — JSON event
```

JSON event örneği:
```json
{ "event": "S", "event_cnt": 12 }
```

## Kullanım Senaryoları

- **Mevcut anahtarı akıllı yapma:** Klasik anahtarın paralel kontağına i3 bağla, anahtar normal çalışır + her basışta SKAPP'e event git
- **Çoklu fonksiyonlu buton:** Tek butona kısa bas → ışık on/off, uzun bas → tüm ışıklar off, çift bas → sahne 1
- **Kapı zili event'i:** Zil butonuna i3 bağla, telefonuna bildirim gelsin
- **Manuel mola tetikleyici:** Masaüstünde bir butona basınca SKAPP'e mola komutu git

## SKAPI Action Şablonları

i3'ün **kendisine** komut gönderme genelde gerek değil (input cihazı). Ama i3'ün event'lerini SKAPP/SKAPI için **giriş kaynağı** olarak kullanabilirsin.

### i3 → SKAPP webhook konfigürasyonu (i3'e bir kez yazılır)
```yaml
id: shelly-i3-webhook-setup
name: "Shelly i3 - SKAPP'e Webhook Bağla"
category: setup
target: shelly-i3
protocol: http
method: GET
url: "http://{ip}/settings/actions?index=0&name={event}&enabled=true&urls[]={skapp_url}"
params:
  - { key: ip, label: "i3 IP", type: ip, required: true }
  - { key: event, label: "Event tipi", type: enum, options: [shortpush_url, longpush_url, double_shortpush_url] }
  - { key: skapp_url, label: "SKAPP webhook URL", type: string, default: "http://192.168.1.50:5000/api/i3-event" }
```

### Event durumunu sorgula (read-only)
```yaml
id: shelly-i3-status
name: "Shelly i3 - Input Durum Oku"
category: smarthome
target: shelly-i3
protocol: http
method: GET
url: "http://{ip}/input/{channel}"
params:
  - { key: ip, label: "Cihaz IP", type: ip, required: true }
  - { key: channel, label: "Kanal (0-2)", type: int, min: 0, max: 2 }
```

## SKAPI içinde i3'ün rolü

i3 normal "aksiyon hedefi" değil — **trigger source**. ESP32-C6 gibi başka bir tetikleyici. Üç farklı kullanım:

1. **i3 → ESP32 (üzerinden) → SKAPP:** i3 bir webhook'u ESP32'ye yollar, ESP32 SKAPP'e iletir
2. **i3 → SKAPP (direkt):** i3 webhook'unu doğrudan SKAPP'in `/api/i3-event` endpoint'ine yolla
3. **i3 → MQTT → SKAPP:** i3 MQTT publish, SKAPP subscribe — daha temiz event-driven mimari
