# Shelly 2.5 (Gen1) — Tam API Komut Listesi

**Model:** SHSW-25
**Nesil:** Gen1
**Özellikler:** **2 röle** (kanal 0 ve 1) + **güç ölçümü her kanalda** + **roller (panjur/perde) modu seçeneği**
**Maks yük:** 10A (2300W) **kanal başına**
**Protokol:** HTTP REST, MQTT, CoAP, Cloud
**Varsayılan port:** 80 (HTTP)

> Shelly 2.5'in iki çalışma modu vardır: **Relay mode** (2 bağımsız röle) veya **Roller mode** (perde/panjur motoru kontrolü).
> İki mod arasında geçiş yapmak için ayar değiştirilir.

---

## 1. Cihaz Bilgileri

### `GET /shelly`
```
http://192.168.1.42/shelly
```

**Cevap:**
```json
{
  "type": "SHSW-25",
  "mac": "...",
  "auth": false,
  "fw": "20230913-...",
  "num_outputs": 2,
  "num_meters": 2,
  "num_rollers": 1,
  "mode": "relay"
}
```

`mode` alanı `relay` veya `roller` olabilir.

### `GET /status`
Tam durum (her iki röle + güç + sıcaklık).

### `GET /settings`
Tam ayarlar.

---

## 2. Mod Seçimi (Relay ↔ Roller)

### Relay moduna geç
```
GET http://192.168.1.42/settings?mode=relay
```

### Roller moduna geç
```
GET http://192.168.1.42/settings?mode=roller
```

⚠️ Mod değişikliği sonrası cihaz kendini yeniden başlatabilir.

# ROLLER MODE (Perde / Panjur Motoru)

## 9. Roller Açma / Kapama / Durdurma

### Aç (yukarı)
```
GET http://192.168.1.42/roller/0?go=open
```

### Kapat (aşağı)
```
GET http://192.168.1.42/roller/0?go=close
```

### Durdur
```
GET http://192.168.1.42/roller/0?go=stop
```

### Belirli pozisyona git (0-100%)
```
GET http://192.168.1.42/roller/0?go=to_pos&roller_pos=50
```
50 = yarı açık.
0 = tam kapalı, 100 = tam açık.

### Manuel offset ile hareket
```
GET http://192.168.1.42/roller/0?go=open&duration=5
```
5 saniye boyunca açma yönüne hareket et.

## 10. Roller Durumu

```
GET http://192.168.1.42/roller/0
```

**Cevap:**
```json
{
  "state": "stop",          // stop / open / close
  "source": "http",
  "power": 0.0,
  "is_valid": true,
  "safety_switch": false,
  "overtemperature": false,
  "stop_reason": "normal",
  "last_direction": "open",
  "current_pos": 50,
  "calibrating": false,
  "positioning": true
}
```

## 11. Kalibrasyon

İlk kullanımda kalibrasyon gerekir (motor pozisyon takibi için).

### Kalibrasyonu başlat
```
GET http://192.168.1.42/roller/0/calibrate
```

Cihaz kendini açıp kapatarak süreleri öğrenir.

## 12. Roller Konfigürasyonu

### Maksimum açılma süresi
```
GET http://192.168.1.42/settings/roller/0?maxtime=20
```
20 saniye sonra motor durur.

### Açma süresi (kalibrasyon sonrası)
```
GET http://192.168.1.42/settings/roller/0?maxtime_open=18&maxtime_close=20
```

### Pozisyon takibi açık/kapalı
```
GET http://192.168.1.42/settings/roller/0?positioning=true
```

### Reverse (yön değiştir)
```
GET http://192.168.1.42/settings/roller/0?swap=true
```

### Power-on davranışı (Roller mode)
```
GET http://192.168.1.42/settings/roller/0?default_state=stop|open|close
```

### Buton modu (Roller mode)
```
GET http://192.168.1.42/settings/roller/0?btn_type=momentary|toggle|detached
```

## 14. Webhook / Actions

Relay mode action isimleri (her kanal):
- `btn_on_url`, `btn_off_url`
- `out_on_url`, `out_off_url`
- `shortpush_url`, `longpush_url`
- `over_power_url`
- `over_temperature_url`

Roller mode action isimleri:
- `roller_open_url`
- `roller_close_url`
- `roller_stop_url`

```
GET http://192.168.1.42/settings/actions?index=0&name=out_on_url&enabled=true&urls[]=http://YOUR-SERVER/on
```

---

## 15. MQTT Konfigürasyonu

### MQTT etkinleştir
```
GET http://192.168.1.42/settings?mqtt_enable=true&mqtt_server=192.168.1.10:1883
```

Relay mode topic'leri:
- `shellies/shellyswitch25-XXX/relay/0` — durum
- `shellies/shellyswitch25-XXX/relay/0/command` — `on`/`off`/`toggle`
- `shellies/shellyswitch25-XXX/relay/0/power`
- `shellies/shellyswitch25-XXX/relay/1/...` (ikinci kanal)

Roller mode topic'leri:
- `shellies/shellyswitch25-XXX/roller/0` — `open`/`close`/`stop`
- `shellies/shellyswitch25-XXX/roller/0/command` — `open`/`close`/`stop`
- `shellies/shellyswitch25-XXX/roller/0/command/pos` — `0-100`
- `shellies/shellyswitch25-XXX/roller/0/pos` — mevcut pozisyon

## Hata Kodları

| Cevap | Anlam |
|---|---|
| `200 OK` + JSON | Başarılı |
| `400` | Yanlış parametre veya yanlış mod (örn. roller mode'da `relay` çağırma) |
| `401` | Auth yanlış |
| `404` | Yanlış endpoint |

---

## Sorun Giderme

- Roller komutu çalışmıyor → mod `roller` mı? `/shelly` ile kontrol et
- Pozisyon yanlış → kalibrasyon yapıldı mı? `/roller/0/calibrate`
- Aşırı ısınma uyarısı → yük çok yüksek olabilir, max güç limitini düşür
- Kanallar ters → `swap=true` ayarı veya buton ters bağlı olabilir
